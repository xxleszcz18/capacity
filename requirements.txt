1. WYMAGANIA ŚRODOWISKOWE
================================================================================

1.1 System operacyjny
    - Windows 10/11, Linux lub macOS (rozwój i produkcja)
    - PowerShell lub bash do uruchamiania skryptów npm

1.2 Oprogramowanie wymagane
    - Node.js >= 18.x (zalecane LTS 20.x lub 22.x)
    - npm >= 9.x (dostarczany z Node.js)
    - Przeglądarka: Chrome, Edge lub Firefox (aktualna wersja)
    - Git (opcjonalnie, do kontroli wersji)

1.3 Zasoby sprzętowe (minimum)
    - RAM: 4 GB (zalecane 8 GB przy dużej bazie i imporcie Excel)
    - Dysk: 500 MB wolnego miejsca (kod + node_modules + baza + backupy)
    - Procesor: dowolny x64 obsługujący Node.js 18+

1.4 Sieć (tryb deweloperski)
    - Port 3001 — serwer API (Express)
    - Port 5173 — klient deweloperski (Vite)
    - Proxy Vite: /api -> http://localhost:3001

9. WYMAGANIA OBLICZENIOWE (CAPACITY)
================================================================================

9.1 Dostępność maszyny [s/tydzień]
    availability = (working_days_year / 52) * shift_time_min * 60 * shifts_per_day * OEE
                   - startup_shutdown_seconds
    Skorygowane o machine_usage maszyny.

9.2 Wolumen tygodniowy
    - annual:   volume / working_weeks_per_year
    - monthly:  (volume * 12) / working_weeks_per_year
    - weekly:   volume (bez przeliczenia)

9.3 Czas wymagany [s/tydzień]
    required = SUM(weekly_volume * (cycle_time_seconds / nests_count)) per operacja

9.4 Obciążenie [%]
    load_percent = round(SUM(required_op / availability_op) * machine_usage * 100)

9.5 Priorytet OEE
    operacja > nadpis maszyny > ustawienia globalne (dni robocze)

9.6 SOP/EOP
    Ułamek roku ogranicza wolumen operacji projektu do miesięcy w zakresie produkcji.

9.7 Profile obliczeń
    - Capacity — domyślny profil parametrów
    - OCU — osobny zestaw parametrów (włączany w ustawieniach administracyjnych)
    - W trybie scenariusza zawsze profil Capacity
